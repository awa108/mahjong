# RULES — 简化四人麻将规则（simple4）

## 牌张
- 万(1-9)/筒(1-9)/条(1-9) 各 ×4 = 108
- 东南西北中发白 ×4 = 28
- 合计 136 张，无花牌

## 胡牌牌型

### 标准型（4 面子 + 1 雀头）
- 面子 = 刻子（3 相同）或顺子（3 连号，仅万/筒/条）
- 雀头 = 对子（2 相同）

### 七对
- 恰好 7 对（14 张）

## 动作

| 动作 | 时机 | 条件 |
|------|------|------|
| 摸 | 自己回合 | 无操作声明 |
| 出 | 自己回合 | 手牌超 14/17/… |
| 吃 | 上家出牌后 | 有构成顺子的 2 张 |
| 碰 | 任意家出牌 | 手里有 2 张相同 |
| 明杠 | 他家出牌 | 手里有 3 张相同 |
| 暗杠 | 自己回合 | 手里有 4 张相同 |
| 补杠 | 自己回合 | 已有碰，摸到第 4 张 |
| 点炮胡 | 他家出牌 | 手牌组成 4 面子 + 1 雀头 |
| 自摸胡 | 自己摸牌后 | 同上 |

## 优先级（多声明冲突时）
胡 > 杠 > 碰 > 吃
若多家同时满足，按座位轮转顺序解析。

## 计分（第一版）

### 基本参数
- 基分 = 1（固定底分，不因番型变动）
- 第一版不分具体番种，所有胡牌牌型得分相同

### 自摸胡
- 赢家：+3 × 基分 = +3
- 其余三家：各 -1 × 基分 = -1
- 总分守恒：+3 - 1 - 1 - 1 = 0

### 点炮胡
- 赢家：+3 × 基分 = +3
- 放炮者：-3 × 基分 = -3
- 其余两家：不扣分（0）
- 总分守恒：+3 - 3 = 0

### 流局
- 所有玩家分数不变
- 服务端 phase 变为 `settled`，不再接受任何游戏动作（PLAY_TILE/CHI/PENG/GANG/HU/PASS）

### 动作拒绝
- 对局结束后（phase: settled），服务端拒绝所有游戏动作，返回错误码 `ILLEGAL_ACTION`，消息 "对局已结束"

### 示例

| 场景 | 赢家变化 | 放炮者变化 | 其余两家 |
|------|---------|-----------|---------|
| 自摸 | +3 | -1 | -1 |
| 点炮 | +3 | -3 | 0 |
| 流局 | 0 | 0 | 0 |

## 扩展预留

后续版本可基于此规则扩展：
- 番种定义（清一色、碰碰胡、混一色等），通过 `checkHu` 的 `HuResult.pattern` 输出计算番数
- 自摸/点炮的基分乘番数
- 庄家连庄加分
- 不同地方规则切换（四川/国标/血战）通过 `Ruleset` 接口实现
